Java String Literals Versus String Objects

20 Mar 2014 on tech

Strings in Java can be declared either literally-

String s = "123";

or as a String instance-

String s = new String("123");

What are the differences between these two strings?

 1package drills;
 2
 3public class Tester {
 4
 5  public static void main(String[] args) {
 6
 7    // Test differences between String v = new String(...)
 8    // and String v = "..."
 9
10    // String literals are stored in a string pool- cached and reused.
11    // String objects are stored in the heap as new objects.                
12
13    // Two string literals with the same characters are the same
14    // in every possible way.
15    String s1 = "abcdef";
16    String s2 = "abcdef";
17    assert(s1 == s2);
18    assert(s1.hashCode() == s2.hashCode());
19    assert(s1.equals(s2) && s2.equals(s1));
20
21    // Two string objects with the same characters are the same
22    // only with hashCode() and equals()- but are different object
23    // instances
24    String s3 = new String("abcdef");
25    String s4 = new String("abcdef");
26    assert(s3 != s4);
27    assert(s3.hashCode() == s4.hashCode());
28    assert(s3.equals(s4) && s4.equals(s3));
29
30    // String literals and String objects are separate string
31    // instances, but hashCodes are all equal
32    assert(s1 != s3 &amp;&amp; s1 != s4);
33    assert(s1.hashCode() == s2.hashCode());
34    assert(s2.hashCode() == s3.hashCode());
35    assert(s3.hashCode() == s4.hashCode());
36
37    // Though separate instances, contents are also still equal
38    assert(s1.equals(s3) && s1.equals(s4));
39    assert(s2.equals(s3) && s2.equals(s4));
40
41    System.out.println("Tests completed");
42  }
43}
44

Summary:

  • Regardless of how strings are declared, for a given character sequence, their hashCodes() and equals() all return the same value.
    • String literals reuse the same string instance if already exists.
    • String objects always create a new instance (unless using the intern method).
