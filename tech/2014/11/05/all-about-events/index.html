All About Events

05 Nov 2014 on tech

Letâ€™s discuss a little bit more about Events. How are Events different from Delegates?

For events-

  • The event keyword is a modifier- to a delegate type, the most frequent being EventHandler for event handling situations.
  • The event keyword allows a delegate type to be exposed in an interface.
  • Events come with add and remove methods, much like get and set methods for properties.
  • Events have a strict method signature (object ok, EventArgs e).

31
32        public void Run()
33        {
34            Random r = new Random();
35            for(int i=0; i<1000; i++)
36            {
37                if (r.Next(100)==1)

39                    ErrorEventHappened(this, new TinyMechErrorArgs(i));
40                }
41            }
42        }

53    class TinyReporter
54    {
55        public void ListenToMachine(TinyMech m)
56        {
57            m.ErrorEventHappened += ReportError;
58        }
59
60        private void ReportError(object sender, EventArgs e)
61        {
62            if (sender is TinyMech)
63            {
64                TinyMechErrorArgs ea = (TinyMechErrorArgs) e;
65                Console.Out.WriteLine("Error on {0}", ea.MechError);
66            }
67        }
